## 1. Give us a URL

We'll ask for a URL to create test cases. The URL has to be publicly accessible.

## 2. Name your project

The second page will render a screenshot of the url you provided. It tests that your URL is accessible. If it's not, you'll see an Octo in a jar.
Now you'd need to give your project a name. We'll propose one similar to the url you provided.
But you can choose your own.

## 3. Select UI test cases to be auto-discovered

At the first set-up, you can select what should we auto-discover:

<Frame caption="Third page of the setup flow - choose your first tests, screenshot 02/2024">
  <img
    src="images/first-steps/auto-discover.png"
    alt="choose your first tests screen, screenshot 10/2023"
  />
</Frame>

**log in**
- this test assesses login functionality, verifying credential accuracy and security, to ensure a seamless and secure user access
- if you choose the log in test, we’ll ask you for test user credentials (test username and a password). The test credentials should fit the URL you provided above

<Frame caption="Third page of the setup flow - insert test user credentials for a log in test, screenshot 02/2024">
  <img
    src="images/first-steps/auto-discover-login.png"
    alt="choose your first tests screen, screenshot 10/2023"
  />
</Frame>

**change language**
- this test checks language switching functionality for smooth and accurate application localization

<Frame caption="Third page of the setup flow - change language test, screenshot 02/2024">
  <img
    src="images/first-steps/auto-discover-change-language.png"
    alt="choose your first tests screen, screenshot 10/2023"
  />
</Frame>

**links**

- this test verifies the accessibility of all links on the page, ensuring they are functional and lead to the correct destinations

<Frame caption="Third page of the setup flow - links test, screenshot 02/2024">
  <img
    src="images/first-steps/auto-discover-links.png"
    alt="choose your first tests screen, screenshot 10/2023"
  />
</Frame>

**only custom**
- note that a reachability test is always active by default. Selecting any test will execute it in conjunction with the
reachability test
- choosing the “only custom” option will run just the reachability test
- additionally, later in the application, you have the flexibility to conduct your own test regardless of the options previously selected

## 5. Enter application & Tutorial

If automatic redirection to our app does not occur after confirming your email, manually log in with the previously set credentials.
Welcome!

Upon logging in, a tutorial will be presented to you. We kindly recommend you **not to skip tutorial**, but it is up to you!

The tutorial provides essential guidance on how to **publish** your test case, created during the sign up process and
which is currently in draft form. Following publication, it will also instruct you on how to request a **test report**, which will lead to the presentation of your test results.
