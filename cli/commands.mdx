# Octomind CLI Commands

The Octomind CLI provides several commands to help you interact with the Octomind API. This guide covers all available commands and their options.

## Execute Tests

Run test cases against a specified URL.

```bash
octomind execute \
  -u, --url <url> \
  [-k, --api-key <key>] \
  [-t, --test-target-id <id>] \
  [-e, --environmentName <name>] \
  [-d, --description <text>] \
  [-g, --tags <list of tags>] \
  [-v, --variables-to-overwrite <variables>] \
  [-j, --json]
```

### Options
- `-u, --url` (required): URL to test
- `-k, --api-key`: Your Octomind API key (optional if configured via `init`)
- `-t, --test-target-id`: Test target ID (optional if configured via `init`)
- `-e, --environmentName`: Environment name (default: "default")
- `-d, --description`: Test description
- `-g, --tags`: Comma separated list of tags for tests to execute
- `-v, --variables-to-overwrite`: JSON object for variables to override for this run e.g. `{ "key": ["v1", "v2"]}`
- `-j, --json`: Output raw JSON response

## Register Private Location

Register a new private location worker. If you use the [private location worker](https://github.com/OctoMind-dev/private-location-worker) it will register itself on startup automatically.

```bash
octomind register-location \
  -n, --name <name> \
  -p, --password <password> \
  -u, --username <user> \
  -a, --address <address> \
  [-k, --api-key <key>] \
  [-j, --json]
```

### Options
- `-n, --name` (required): Location name
- `-p, --password` (required): Proxy password
- `-u, --username` (required): Proxy user
- `-a, --address` (required): Location address (format: IP:PORT)
- `-k, --api-key`: Your Octomind API key (optional if configured via `init`)
- `-j, --json`: Output raw JSON response

## Unregister Private Location

Remove a registered private location worker. If you use the [private location worker](https://github.com/OctoMind-dev/private-location-worker) it will unregister itself when going offline automatically.

```bash
octomind unregister-location \
  -n, --name <name> \
  [-k, --api-key <key>] \
  [-j, --json]
```

### Options
- `-n, --name` (required): Location name
- `-k, --api-key`: Your Octomind API key (optional if configured via `init`)
- `-j, --json`: Output raw JSON response

## Create Environment

Create a new environment for a test target.

```bash
octomind create-environment \
  -n, --name <name> \
  -d, --discovery-url <url> \
  [-k, --api-key <key>] \
  [-t, --test-target-id <id>] \
  [--test-account-username <username>] \
  [--test-account-password <password>] \
  [--test-account-otp-initializer-key <key>] \
  [--basic-auth-username <username>] \
  [--basic-auth-password <password>] \
  [--private-location-name <name>] \
  [--additional-header-fields <fields>] \
  [-j, --json]
```

### Options
- `-n, --name` (required): Environment name
- `-d, --discovery-url` (required): Discovery URL
- `-k, --api-key`: Your Octomind API key (optional if configured via `init`)
- `-t, --test-target-id`: Test target ID (optional if configured via `init`)
- `--test-account-username`: Test account username
- `--test-account-password`: Test account password
- `--test-account-otp-initializer-key`: OTP initializer key for test account
- `--basic-auth-username`: Basic auth username
- `--basic-auth-password`: Basic auth password
- `--private-location-name`: Private location name
- `--additional-header-fields`: Additional header fields (JSON string)
- `-j, --json`: Output raw JSON response

## Start Private Location

Start a private location worker.

```bash
octomind start-private-location \
  [-n, --name <name>] \
  [-u, --username <username>] \
  [-p, --password <password>] \
  [-l, --host-network] 
```

### Options
- `-n, --name`: Location name, default is "default-plw"
- `-u, --username`: Proxy user, default is random username
- `-p, --password`: Proxy password, default is random password
- `-l, --host-network`: Use host network, default is false. When set you can use localhost directly

## Stop Private Location

Stop a running private location worker.

```bash
octomind stop-private-location
```

## Create Discovery

```bash

  -n, --name <name> \
  -p, --prompt <prompt> \

  [-t, --test-target-id <id>] \

  [--external-id <id>] \
  [--assigned-tag-ids <ids>] \
  [--folder-id <id>] \
  [-j, --json]
```

### Options
- `-n, --name` (required): Discovery name
- `-p, --prompt` (required): Discovery prompt
- `-k, --api-key`: Your Octomind API key (optional if configured via `init`)
- `-t, --test-target-id`: Test target ID (optional if configured via `init`)

- `--external-id`: External identifier
- `--assigned-tag-ids`: Comma-separated list of tag IDs
- `--folder-id`: Folder ID
- `-j, --json`: Output raw JSON response

## Debug

Run test cases locally. This will use a local browser controlled by playwright to run the test cases.
It will still use the configured setup include the proxy setting for the environment.
You can bypass the proxy by using "http://localhost:port" as the URL.

```bash
octomind debug \
  -u, --url <url> \
  [-k, --api-key <key>] \
  [-i, --id <uuid>] \
  [-e, --environmentId <uuid>] \
  [-a, --testTargetId <uuid>] \
  [--headless] \
  [--persist]
```

### Options
- `-u, --url` (required): URL the tests should run against
- `-k, --api-key`: Your Octomind API key (optional if configured via `init`)
- `-i, --id`: ID of the test case you want to run (if not provided, will run all test cases in the test target)
- `-e, --environmentId`: ID of the environment you want to run against (if not provided, will run against the default environment)
- `-a, --testTargetId`: ID of the test target of the test case
- `--headless`: Run headless without the UI of Playwright and the browser
- `--persist`: Write Playwright config and files to current directory (you can then run 'npx playwright test' to run them again)

## Output Formats

By default, the CLI provides formatted text output for better readability. Add the `--json` flag to any command to get the raw JSON response instead. This is useful for scripting or when you need to process the output programmatically.
