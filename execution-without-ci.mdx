If you don't have pipelines or you just want to manually trigger the test execution, you can do it with the following cURL command:

```
curl -w --request https://app.octomind.dev/api/v2/execute \
--header 'Content-Type: application/json' \
--header 'x-api-key: <YOUR_API_KEY>' \
--data \
"{\
    \"url\": \"<YOUR_WEBSITE_URL_TO_TEST_AGAINST>\",\
    \"testTargetId\": \"<YOUR_TEST_TARGET>\",\
    \"context\":{\
        \"source\": \"manual\",\
        \"description\": \"manual trigger\",\
        \"triggeredBy\": {\
            \"type\": \"USER\",\
            \"userId\": \"<YOUR_USER_ID>\"\
    }}}"
```

You can see how to set up your `API key` and get your `test target id` and `user id` [below](/manual-execution#create-an-api-key).

`YOUR_WEBSITE_URL_TO_TEST_AGAINST` should be the base url that you provided when you first visited the Octomind webpage.

The command will return a response with link to your test report, looking something like this:

```
{"testReportUrl":"https://app.octomind.dev/testreports/<some-id>", ...}
```

On the above mentioned link, you can find the test report containing all test results of your execution.
