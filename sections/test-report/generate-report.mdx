When generating report, we run all the published test cases and put test result of each of them into one report.
Report is successful if each individual published test case run is successful.

Once you clicked the generate report button, which can be found in multiple places within the app, you can select one of the following four options:

- [in app](/sections/test-report/generate-report/#in-app)
- [in app via terminal](/sections/test-report/generate-report/#in-app-via-terminal)
- [via CI](/sections/test-report/generate-report/#via-ci)
- [locally](/sections/test-report/generate-report/#locally)

# in app

By selecting this option, your report generation will start immediately.
Results will be visible within the application and statuses will update accordingly.

# in app via terminal

By selecting this option, pop up will appear with the command you to **copy** and **execute it within the terminal.**
After execution, navigate to the application's **test reports** section and your results will be visible.

Additionally, you will need to use an **API key** and incorporate it into the process. [Here]() is how you can get one.
